## Position 5: AI Agent Builder

### Persona
**Kevin Zhao**, Senior AI Engineer, 스타트업 (35명)
- LangChain + CrewAI로 고객 지원 자동화 에이전트 구축 중
- 매주 2-3개 새로운 MCP 서버/API를 에이전트에 연결해 테스트
- 도구 하나 잘못 고르면 에이전트 전체가 망가지는 걸 뼈저리게 경험함
- GitHub, Discord, Reddit을 뒤져가며 "이 MCP 서버 실제로 작동하냐" 확인하는 데 시간 낭비

### Clarvia를 왜 쓰겠는가
CrewAI로 코드 리뷰 에이전트를 만드는 중인데, GitHub MCP 서버가 3개나 있다. 어떤 게 가장 안정적이고 에이전트 호환성이 좋은지 모르겠다. Clarvia에서 "github mcp server" 검색하고, Agent Compatibility 점수 비교해서 가장 높은 걸 고르겠다. 비교 기능으로 3개 나란히 놓고 판단.

### 좋은 점
1. **Agent Compatibility 차원이 진짜 핵심이다.** 다른 데서는 "이 도구 좋아요" 정도인데, Clarvia는 에이전트가 실제로 잘 연결할 수 있는지를 점수로 보여준다. 이거 하나만으로도 북마크할 가치 있다.
2. **의도 기반 추천이 놀랍게 잘 맞는다.** "I want to automate code reviews with AI agents"라고 치면 관련 도구가 쭉 나온다. 키워드 검색보다 훨씬 빠르게 원하는 걸 찾는다.
3. **12,800개 도구가 한 곳에.** 전에는 awesome-mcp-servers GitHub, Reddit r/mcp, 각 프레임워크 docs를 다 뒤져야 했는데, 여기서 한 번에 해결된다.

### 불만/아쉬운 점
1. **프레임워크별 호환성 정보가 없다.** Agent Compatibility가 일반적인 점수인데, 내가 알고 싶은 건 "이 MCP 서버가 LangChain에서 잘 되냐, CrewAI에서 잘 되냐"다. LangChain에서는 잘 되는데 AutoGen에서는 안 되는 경우가 실제로 많다. 프레임워크별 호환성 태그나 필터가 필요하다.
2. **실제 사용 코드 스니펫이 없다.** 도구를 찾았으면 바로 연결하고 싶은데, 설치 방법이나 연결 코드가 Clarvia에는 없다. 결국 GitHub README로 다시 가야 한다. 도구 찾기 -> 연결하기 사이의 간극이 크다.
3. **도구 안정성/다운타임 이력이 안 보인다.** AEO 점수가 높아도 실제로 서버가 자주 죽거나 응답이 느리면 에이전트가 멈춘다. 업타임 모니터링이나 최근 이슈 히스토리 같은 게 있어야 프로덕션에 쓸 수 있다.

### 킬러 피처
**"Agent Playground" - 도구를 Clarvia 안에서 바로 테스트할 수 있는 샌드박스.** 도구 상세 페이지에서 "Try it" 버튼 누르면 실제 API 호출이나 MCP 서버 연결을 시뮬레이션할 수 있으면, 매일 쓴다. 지금은 도구 찾고 -> 로컬에 설치하고 -> 테스트하고 -> 안 되면 다른 거 찾고... 이 루프가 너무 길다.

### 점수: 6.5/10
도구 발견(discovery)은 최고 수준이지만, 발견 이후의 워크플로(테스트, 연결, 모니터링)가 비어 있다. 지금은 "좋은 카탈로그"이고, "필수 도구"까지는 아직 거리가 있다.

## Position 6: Platform Engineer

### Persona
**Sarah Kim**, Staff Platform Engineer, 핀테크 회사 (450명)
- 내부 개발자 200+명이 쓰는 IDP(Internal Developer Platform) 관리
- AI 코파일럿/에이전트를 사내 개발 워크플로에 도입하라는 CTO 지시 받은 상태
- 보안 심사, 컴플라이언스, SOC2 감사를 항상 신경 써야 함
- "개발자들이 아무 MCP 서버나 갖다 쓰는 거 막아야 한다"가 현재 과제

### Clarvia를 왜 쓰겠는가
사내에서 AI 에이전트에 연결할 도구를 승인 리스트로 관리해야 한다. Clarvia의 AEO 스캔으로 각 도구의 API 접근성, 데이터 구조화, 신뢰 신호를 한 번에 평가하고, 일정 점수 이상만 내부 승인 목록에 올리는 프로세스를 만들려고 한다.

### 좋은 점
1. **Trust Signals 차원이 보안 심사의 첫 단계로 쓸 만하다.** 라이선스, 문서화 수준, 커뮤니티 활동을 한 눈에 본다. 전에는 각 도구마다 GitHub 가서 일일이 확인했는데 시간을 크게 줄인다.
2. **비교 기능이 의사결정 문서 만들기에 좋다.** 4개까지 비교해서 스크린샷 찍으면 바로 내부 승인 문서에 붙인다. SNS 공유 카드도 의외로 쓸모 있다 -- Slack에 공유하면 동료들이 바로 본다.
3. **AEO 점수가 객관적 기준선 역할을 한다.** "이 도구 왜 안 돼요?"라는 개발자 질문에 "AEO 점수 40점이라서"라고 데이터로 답할 수 있다. 감정 없이 판단 근거를 댈 수 있는 게 큰 장점.

### 불만/아쉬운 점
1. **팀/조직 기능이 전무하다.** 나 혼자 스캔하고 비교하는 건 되는데, 팀 단위로 승인 목록 관리하거나 스캔 결과를 공유하는 게 안 된다. 승인/거부 워크플로, 팀 대시보드가 없으면 엔터프라이즈에서는 쓸 수 없다. 이게 가장 큰 벽이다.
2. **보안 심층 분석이 부족하다.** Trust Signals가 있긴 한데, 실제 보안 팀이 원하는 건 의존성 취약점 스캔, 데이터 흐름 분석, 권한 범위(scope) 확인이다. 지금 점수로는 "참고용"이지 "근거"가 되지 못한다.
3. **API가 없다.** 내부 IDP에 Clarvia 데이터를 연동하고 싶은데 API가 없으니까 수동으로 해야 한다. CI/CD 파이프라인에서 "새 도구 추가 시 자동으로 AEO 체크" 같은 걸 못 만든다. 이건 플랫폼 엔지니어한테 치명적.

### 킬러 피처
**Clarvia API + Policy Engine.** REST API로 AEO 점수를 조회하고, 조직별 정책(예: "Trust Signal 70점 이하 자동 거부", "OnChain 보너스 없으면 경고")을 설정할 수 있으면 내부 도구 거버넌스 시스템에 바로 통합한다. 이거 나오면 유료로 쓴다.

### 점수: 4.5/10
개인 도구로는 괜찮지만 조직에서 쓰기엔 너무 이르다. API 없음 + 팀 기능 없음 + 보안 심층 분석 없음의 3중 결핍. 지금은 내가 개인적으로 도구 사전 조사할 때만 가끔 쓰는 수준.

## Position 7: Tech Lead / CTO

### Persona
**Jason Park**, CTO, B2B SaaS 스타트업 (28명)
- 풀스택 개발 + 아키텍처 의사결정 + 채용까지 다 하는 중
- AI 에이전트로 개발 생산성 올리고 싶은데, 도구가 너무 많아서 뭘 써야 할지 모르겠음
- 시간이 없어서 30분 안에 판단하고 넘어가야 함
- 잘못된 기술 선택으로 3개월 날린 경험 있음

### Clarvia를 왜 쓰겠는가
"우리 제품에 AI 에이전트 기반 고객 지원을 넣자"라고 결정했는데, Zendesk MCP, Intercom API, 자체 구축 중 뭘 할지 고민이다. Clarvia에서 관련 도구들 검색하고, AEO 점수로 빠르게 상위 3개 추리고, 비교해서 기술 의사결정 내리겠다.

### 좋은 점
1. **의사결정 속도가 빨라진다.** 트렌딩 + 리더보드 + 비교를 조합하면, 예전에 2-3일 걸리던 도구 리서치를 30분에 끝낼 수 있다. CTO한테 시간 절약은 돈이다.
2. **AEO 점수의 5개 차원이 체계적이다.** API 접근성, 데이터 구조화, 에이전트 호환성, 신뢰, 온체인 -- 이 프레임워크 자체가 "AI 시대에 좋은 도구란 무엇인가"에 대한 답을 제시한다. 내가 팀에 기술 선택 기준을 설명할 때 이 프레임워크를 그대로 쓴다.
3. **트렌딩이 시장 감각을 유지시켜준다.** 주간 인기 도구를 훑어보면 지금 업계가 어디로 가고 있는지 감이 온다. 뉴스레터보다 실질적이다.

### 불만/아쉬운 점
1. **실사용 후기/케이스 스터디가 없다.** 점수만 보고 도구를 선택하기엔 리스크가 크다. "이 도구를 실제로 누가 어떻게 쓰고 있는지"가 없다. 점수 70이어도 우리 유즈케이스에 안 맞을 수 있다. 최소한 사용자 리뷰나 "이런 회사가 쓰고 있다" 정도는 있어야 한다.
2. **카테고리/유즈케이스 큐레이션이 약하다.** 12,800개 도구가 있는데 "고객 지원 자동화에 좋은 도구 top 5" 같은 큐레이션이 없다. 검색해서 하나하나 비교하는 건 CTO가 할 일이 아니다. 유즈케이스별 가이드가 필요하다.
3. **가격/비용 정보가 없다.** 도구의 기술적 품질은 알겠는데, 무료인지, 월 얼마인지, 사용량 제한이 뭔지를 모른다. 기술 의사결정은 항상 비용과 같이 간다. 이게 빠져 있으면 결국 각 도구 사이트를 또 방문해야 한다.

### 킬러 피처
**"Stack Planner" - 유즈케이스 기반 도구 스택 추천.** "고객 지원 에이전트 구축"이라고 입력하면, 필요한 도구 조합(LLM + MCP 서버 + 벡터DB + 모니터링)을 한 번에 추천해주는 것. 개별 도구 비교가 아니라 "이 조합이 최적"이라는 답을 주면, 매번 새 프로젝트 시작할 때 첫 번째로 들어온다.

### 점수: 5.5/10
초기 리서치 도구로는 쓸 만하지만, 의사결정까지 가려면 부족한 게 많다. 점수 + 트렌드는 좋은데, 사용 후기 + 비용 + 큐레이션이 빠져서 "도구 발견의 시작점"으로만 기능한다. 결정은 결국 다른 데서 한다.

## Position 8: Open Source Maintainer

### Persona
**Alex Rivera**, 오픈소스 개발자, 프리랜서
- `mcp-postgres`라는 PostgreSQL MCP 서버 유지 중 (GitHub stars 2.4k)
- 매달 이슈 30-40개 처리, PR 리뷰, 릴리스 관리
- 프로젝트의 신뢰도와 품질을 입증해서 더 많은 유저/기여자를 끌어오고 싶음
- 도구 디렉토리/마켓플레이스에 등록되는 게 노출에 핵심이라는 걸 알고 있음

### Clarvia를 왜 쓰겠는가
mcp-postgres의 AEO 점수를 확인하고, 높은 점수를 받아서 README에 배지를 달고 싶다. 리더보드에 올라가면 유저 유입이 늘 거고, 점수가 낮은 부분을 개선 가이드 삼아 프로젝트를 더 좋게 만들겠다.

### 좋은 점
1. **AEO 배지가 신뢰의 상징이 될 수 있다.** README에 "AEO Score: 87/100" 배지 달면, 처음 보는 사람이 "아 이건 검증된 거구나"라고 느낀다. 기존에는 stars 수밖에 보여줄 게 없었다. 제3자 품질 인증 역할을 해준다.
2. **개선 포인트가 명확하다.** 5개 차원별 점수를 보면 "아, 데이터 구조화에서 깎였구나. JSON Schema 추가해야겠다"처럼 구체적인 액션 아이템이 나온다. 무료 품질 감사를 받는 셈이다.
3. **서비스 등록 기능이 있다.** MCP 서버를 직접 등록할 수 있으니까, 자동 크롤링에만 의존하지 않아도 된다. 내 프로젝트가 누락될 걱정이 줄었다.

### 불만/아쉬운 점
1. **점수 산정 기준이 불투명하다.** "Agent Compatibility: 72"라고만 나오는데, 왜 72인지, 100이 되려면 뭘 해야 하는지가 안 보인다. 세부 채점 루브릭을 공개해줘야 한다. 지금은 블랙박스라서 점수를 올리고 싶어도 감이 안 잡힌다.
2. **메인테이너 대시보드가 없다.** 내 도구가 검색에 몇 번 노출됐는지, 비교에 몇 번 올라갔는지, 트렌딩에 들어갔는지 같은 통계가 전혀 없다. 등록만 하고 끝이면 동기 부여가 안 된다. npm 같은 다운로드 카운터라도 있어야 한다.
3. **점수 재스캔/업데이트 주기가 불명확하다.** 내가 프로젝트를 개선해도 점수가 언제 반영되는지 모르겠다. 수동으로 재스캔 요청하는 것도 가능한지 모르겠다. "개선 -> 확인" 피드백 루프가 끊겨 있다.

### 킬러 피처
**Maintainer Analytics + Rescan Trigger.** 내 프로젝트의 검색 노출 수, 비교 등장 수, 배지 클릭 수를 보여주는 대시보드 + 코드 변경 후 즉시 재스캔 요청 버튼. 이 두 개가 합쳐지면 "Clarvia에서 점수 올리기 = 프로젝트 품질 올리기" 선순환이 만들어진다. 매주 확인하러 올 것이다.

### 점수: 6/10
배지와 점수 시스템의 컨셉은 훌륭하다. 하지만 점수 기준이 불투명하고, 개선 후 확인하는 루프가 없고, 메인테이너를 위한 인센티브(통계, 노출)가 부족하다. 지금은 배지 한 번 달고 끝인 원나잇 관계. 지속적으로 돌아오게 만들 고리가 없다.
