## 1. 프로젝트 출시 준비 상태

### 핵심 인사이트 (3줄 요약)
- AEO는 2026년 최대 마케팅 키워드 중 하나이며, Gartner는 기존 검색량이 25% 감소할 것으로 전망. AI 에이전트 시대의 "발견 가능성"은 새로운 SEO.
- MCP 생태계는 폭발적 성장 중 (공식 레지스트리 6,400+ 서버, 전체 생태계 17,000+). 그러나 "도구 품질 스코어링"에 집중하는 플레이어는 아직 없다.
- Clarvia의 "AEO 스캐너"는 기존 AEO 도구(HubSpot AEO Grader, Source Grader 등)가 "웹사이트의 AI 검색 가시성"에 집중하는 것과 달리, "API/서비스의 AI 에이전트 발견 가능성"이라는 새로운 각도를 잡고 있다.

### 상세 분석

#### 시장 타이밍: 매우 적절

**AEO 트렌드**
- 2026년은 AEO(Answer Engine Optimization)에서 AAIO(Agentic AI Optimization)로의 전환기. AI 에이전트가 단순 검색이 아니라 자율적으로 도구를 선택하고 실행하는 시대.
- HubSpot이 AEO Grader를 출시하며 "AEO"라는 용어가 메인스트림화. 이는 Clarvia의 교육 비용을 줄여준다.
- AI 추천 트래픽이 2024년 7월~2025년 2월 사이 10배 이상 급증 (미국 기준).

**MCP 생태계**
- MCP 서버 다운로드: 2024.11 10만 -> 2025.4 800만+ (80배 성장)
- 2025.12 Anthropic이 MCP를 Linux Foundation 산하 AAIF에 기부. OpenAI, Microsoft, Google, Amazon 모두 채택.
- 2026 로드맵 핵심: MCP Server Cards (`.well-known/mcp/server-card.json`), 원격 서버 디스커버리, 레지스트리 표준화.
- **핵심 기회**: MCP Server Cards 표준이 아직 확정 전(SEP-1649, SEP-1960). Clarvia가 이 표준을 조기 채택하고 스캐너에 반영하면 표준과 함께 성장 가능.

#### 경쟁/유사 서비스 매핑

| 서비스 | 핵심 기능 | 도구 수 | Clarvia와의 차이 |
|--------|----------|---------|-----------------|
| **mcp.so** | MCP 서버 카탈로그 + 플레이그라운드 | 17,186+ | 디렉토리/테스팅. 스코어링 없음 |
| **Glama.ai** | MCP 디렉토리 + 호스팅 게이트웨이 | 9,000+ | 호스팅 중심. AEO 스코어 없음 |
| **Smithery.ai** | MCP 서버 등록/배포/관리 | 2,880+ | DevOps 중심. 품질 평가 없음 |
| **Toolify.ai** | 범용 AI 도구 디렉토리 | 다수 | 일반 AI 도구. MCP/API 특화 아님 |
| **HubSpot AEO Grader** | 브랜드 AI 가시성 점수 | N/A | 웹사이트 브랜드용. API/도구 아님 |
| **Source Grader** | 웹사이트 AI 에이전트 가독성 점수 | N/A | 웹페이지 전용. API 스캔 없음 |
| **AEO Agent AI** | 웹사이트 AEO 최적화 | N/A | 콘텐츠 최적화. 도구 발견 아님 |

#### Clarvia의 차별점

1. **유일한 "AI Engine Optimization" 스캐너**: 기존 AEO = 웹사이트 콘텐츠의 AI 검색 가시성. Clarvia = API/서비스/MCP 서버의 AI 에이전트 발견+사용 가능성. 카테고리 자체가 다르다.
2. **스코어링 + 개선 가이드**: 디렉토리(목록)가 아니라 진단 도구. "당신의 MCP 서버는 72점, 이렇게 고치면 90점" 형태.
3. **의도 기반 추천**: 단순 카테고리 검색이 아닌, "이런 작업을 하고 싶다" -> 최적 도구 추천.
4. **크로스 카테고리 인덱싱**: MCP 서버 + API + CLI + Skills를 통합 인덱싱 (12,800+). 다른 서비스는 MCP만 또는 API만.

### 액션 아이템
1. **MCP Server Cards 표준 조기 채택**: SEP-1649 스펙 구현하고, Clarvia 자체 서버에도 적용. "우리가 스캔하는 기준을 우리가 먼저 통과" 데모.
2. **5개 인기 MCP 서버 케이스 스터디**: Supabase MCP, GitHub MCP, Notion MCP 등을 스캔하고 점수+개선 가이드를 공개 콘텐츠로 만들기.
3. **"AEO" 용어 교육 콘텐츠**: "AEO for APIs" 또는 "AI Engine Optimization vs Answer Engine Optimization" 블로그 포스트로 카테고리 정의.
4. **경쟁사 차별화 페이지**: "Clarvia vs mcp.so vs Smithery" 비교 랜딩.

## 2. 첫 무료 사용자 유치 전략

### 핵심 인사이트 (3줄 요약)
- Product Hunt + Hacker News 동시 공략이 dev tool의 표준 런치 플레이북. PH는 인지도, HN은 기술적 신뢰도.
- MCP 생태계는 빠르게 성장 중이지만 커뮤니티가 아직 밀집되어 있어(MCP Discord, GitHub Discussions) 타깃팅이 매우 효율적.
- "자체 AEO 점수 높은 것을 데모"하는 것은 가장 강력한 신뢰 구축 방법 (dog-fooding).

### 상세 분석

#### 채널별 전략 및 우선순위

##### Tier 1: 최우선 (출시 주)

**1. Product Hunt 런치** | 예상 효과: 1일 2,000-5,000 방문
- **타이밍**: 화~목요일 오전 12:01 AM PST 정각 게시
- **사전 준비 (4주 전)**:
  - PH 커뮤니티 활동 시작 (다른 제품 업보트, 의미 있는 댓글)
  - 헌터 섭외: AI/dev tool 분야 탑 헌터에게 DM (Kevin William David, Chris Messina 등)
  - 티저: "Coming soon" 페이지로 사전 구독자 모집
- **런치 데이**:
  - 메이커 첫 댓글: "왜 만들었는지" 스토리 + 데모 GIF
  - 모든 댓글에 10분 이내 답변
  - 특별 오퍼: "Product Hunt 사용자 첫 스캔 결과 프리미엄 리포트 무료"
- **비주얼**: 스캔 점수 애니메이션 GIF, before/after 비교 이미지
- **키 메시지**: "SEO가 검색엔진용이었다면, AEO는 AI 에이전트용. 당신의 API는 몇 점?"

**2. Hacker News (Show HN)** | 예상 효과: 500-2,000 방문
- **타이밍**: PH 런치와 같은 주, 화~목 오전 8-10 AM PST
- **제목 공식**: `Show HN: Clarvia - I scanned 12K+ MCP servers and APIs for AI agent discoverability`
  - 숫자 포함, 기술적 흥미 유발, 겸손한 톤
- **첫 댓글**: 기술적 접근법 설명 (스코어링 알고리즘, 어떤 요소를 평가하는지)
- **핵심**: HN은 기술적 깊이를 원한다. 마케팅 톤 절대 금지.
- **초기 8-10 업보트**: 사전에 빌더 네트워크에 링크 공유 (조작 아닌 자연 공유)

**3. MCP 생태계 직접 참여** | 예상 효과: 핵심 얼리어답터 50-100명
- **MCP Discord**: 생태계 논의에 참여. "MCP 서버 품질 개선" 관점 공유.
- **GitHub Discussions** (modelcontextprotocol/modelcontextprotocol): SEP-1649 (Server Cards) 논의에 Clarvia 관점 기여.
- **MCP Registry 팀 접촉**: "레지스트리에 등록된 서버의 품질 점수를 Clarvia가 제공"하는 파트너십 제안.
- **인기 MCP 서버 메인테이너에게 DM**: "당신의 서버를 스캔했더니 82점. 이렇게 하면 95점 가능" 개인 리포트 전달.

##### Tier 2: 2-4주차

**4. Dev 커뮤니티 콘텐츠** | 예상 효과: 월 500-1,000 유기적 방문
- **Dev.to / Hashnode**: "AEO란 무엇인가: AI 에이전트 시대의 새로운 SEO" 시리즈
- **Reddit r/programming, r/MachineLearning**: 기술 토론 참여 (스팸 금지, 가치 제공)
- **X/Twitter**:
  - 일일 인사이트: "오늘 스캔한 MCP 서버 중 상위 10%의 공통점" 같은 데이터 기반 트윗
  - 스레드: "12,800개 도구를 스캔하고 발견한 5가지 패턴"
  - 인기 MCP/AI 계정 태그 + 토론

**5. SEO/AEO 자체 시범 (Dog-fooding)** | 예상 효과: 장기 SEO 트래픽 기반
- Clarvia 자체를 AEO 최적화. llms.txt 구현, 구조화된 데이터, .well-known/mcp 엔드포인트.
- "Clarvia의 자체 AEO 점수: 96/100. 이렇게 달성했다" 케이스 스터디.
- AI 검색 (Perplexity, ChatGPT Search)에서 "MCP server quality" 검색 시 Clarvia가 나오도록 최적화.

##### Tier 3: 1-2개월 후

**6. 인플루언서/개발자 파트너십** | 예상 효과: 건당 500-2,000 방문
- AI/MCP 유튜버, 블로거에게 "당신의 MCP 서버를 Clarvia로 분석" 콘텐츠 협업 제안.
- 파트너 MCP 서버에 "AEO Optimized by Clarvia" 배지 부여.
- Anthropic DevRel, OpenAI DevRel 접촉: MCP 에코시스템 개선 도구로 Clarvia 소개.

## 3. 무료 사용자 데이터 활용 성장 전략

### 핵심 인사이트 (3줄 요약)
- 스캔 데이터 자체가 가장 가치 있는 자산. 12,800+ 도구의 AEO 점수 분포, 카테고리별 평균, 트렌드를 공개하면 업계 기준(benchmark)이 된다.
- 배지 시스템은 검증된 바이럴 루프. CI 도구들(Travis CI, Codecov)이 README 배지로 폭발 성장한 모델을 AEO 점수에 적용.
- "비교 카드"와 "트렌딩 리포트"는 자연스러운 공유 트리거. 개발자들은 자신의 도구가 경쟁자보다 높은 점수를 받으면 자발적으로 공유한다.

### 상세 분석

#### 수집할 데이터

| 데이터 유형 | 수집 방법 | 활용 |
|-----------|----------|------|
| 스캔 요청 (URL, 도구 유형) | 스캔 API 로그 | 인기 도구 파악, 트렌드 분석 |
| AEO 점수 분포 | 스캔 결과 집계 | 업계 벤치마크 공개 |
| 검색 쿼리 (의도 기반) | 검색 로그 | 추천 알고리즘 개선, 수요 파악 |
| 비교 패턴 (A vs B) | 비교 기능 로그 | 경쟁 관계 매핑, 대체 도구 추천 |
| 체류시간/이탈률 | 프론트엔드 애널리틱스 | UX 개선, 핵심 기능 파악 |
| 배지 임베드 위치 | 배지 요청 로그 | 역링크 트래킹, 도달 범위 측정 |
| 점수 변화 이력 | 주기적 재스캔 | 개선 추적, 알림 트리거 |

#### 데이터 기반 제품 개선

1. **스코어링 알고리즘 정교화**
   - 실제 AI 에이전트 사용 데이터와 AEO 점수의 상관관계 분석
   - "점수가 높은 도구가 실제로 AI 에이전트에 더 잘 선택되는가?" 검증
   - 카테고리별 가중치 자동 조정

2. **추천 정확도 향상**
   - 검색 쿼리 패턴으로 의도(intent) 분류 모델 학습
   - "데이터베이스 연결" 검색 -> Supabase MCP, Neon MCP, PlanetScale API 등 컨텍스트 기반 추천
   - 클릭률/채택률 기반 A/B 테스트

3. **벤치마크 리포트 자동 생성**
   - 월간 "State of AEO" 리포트: 전체 점수 분포, 카테고리 Top 10, 최대 개선 도구
   - 이 리포트 자체가 SEO 콘텐츠이자 PR 소재

#### 네트워크 효과 구축

**배지 -> 역링크 루프**
```
도구 개발자가 Clarvia 스캔
→ 높은 점수 획득 (AEO Score: 92/100)
→ README/문서에 Clarvia 배지 임베드
→ 배지 클릭 → 다른 개발자가 Clarvia 방문
→ 자신의 도구도 스캔
→ 반복
```

- **배지 디자인**: shields.io 스타일. `![AEO Score](https://clarvia.art/badge/tool-name.svg)`
- **배지 레벨**: Gold (90+), Silver (70-89), Bronze (50-69). 레벨이 올라가면 공유 욕구 증가.
- **참고 사례**: Codecov 배지가 GitHub README에 퍼지며 서비스 인지도 폭발. G2 배지가 SaaS 마케팅 표준이 된 것도 동일 메커니즘.

**비교 카드 -> 소셜 공유 루프**
```
개발자가 "내 도구 vs 경쟁 도구" 비교
→ Clarvia가 비교 카드 생성 (OG 이미지 자동)
→ 개발자가 X/Twitter에 비교 카드 공유
→ "우리 도구가 92 vs 71로 앞선다"
→ 토론/리트윗
→ 양쪽 팬이 Clarvia 방문
```

**리더보드 -> 경쟁 심리 루프**
```
카테고리별 AEO 리더보드 공개
→ "Database MCP Top 10" 등
→ 도구 개발자가 순위 올리기 위해 개선
→ 재스캔 → Clarvia 재방문
→ 순위 변동 시 소셜 공유
```

#### 바이럴 루프 설계

1. **스캔 결과 공유 URL**
   - 모든 스캔 결과에 고유 URL + OG 이미지 자동 생성
   - 예: `clarvia.art/scan/supabase-mcp` -> 점수, 차트, 개선 포인트가 담긴 공유 가능한 페이지
   - 원클릭 트위터/LinkedIn 공유 버튼

2. **주간/월간 트렌딩 리포트 뉴스레터**
   - "이번 주 가장 많이 스캔된 도구 Top 10"
   - "최대 점수 상승 도구"
   - "새로 인덱싱된 도구"
   - 뉴스레터 구독 -> 자체 이메일 리스트 구축

3. **"내 점수 올리기" 가이드**
   - 스캔 결과 페이지에 "점수 올리기 가이드" 직접 링크
   - 가이드 실행 후 재스캔 유도 -> 재방문 + 체류시간 증가

#### 커뮤니티 빌딩

1. **기여자 프로그램**
   - "Clarvia Contributor": 새 도구를 등록/인덱싱에 기여하면 프로필 배지
   - 상위 기여자에게 프리미엄 기능 무료 제공
   - 오픈소스 스코어링 모듈 공개 -> 커뮤니티 기여 유도

2. **도구 등록 셀프서비스**
   - 도구 개발자가 직접 자기 도구를 등록하는 경로 제공
   - 등록 -> 자동 스캔 -> 점수 부여 -> 배지 제공
   - 이 과정 자체가 온보딩 퍼널

3. **Discord/Slack 커뮤니티**
   - "AEO 개선 Q&A" 채널
   - 주간 "AEO Clinic": 도구 개발자가 점수 개선을 위해 질문

### 액션 아이템
1. **W1**: 스캔 결과 공유 URL + OG 이미지 자동 생성 구현
2. **W2**: shields.io 스타일 배지 시스템 구축 (Gold/Silver/Bronze)
3. **W3**: 비교 카드 OG 이미지 자동 생성
4. **W4**: 뉴스레터 구독 시스템 + 첫 "State of AEO" 리포트 초안
5. **W6**: 기여자 프로그램 베타, 도구 셀프 등록 오픈
6. **W8**: 데이터 기반 추천 알고리즘 v2

### 타임라인
- **월 1**: 배지 + 공유 URL + OG 이미지 (바이럴 인프라)
- **월 2**: 뉴스레터 + 리더보드 + 비교 카드 공유

## 4. 유료화 전략

### 핵심 인사이트 (3줄 요약)
- Dev tool SaaS 프리미엄 전환율 벤치마크: 2-5% (일반), 8-15% (상위). Clarvia가 1,000 무료 사용자 확보 시 20-50 유료 사용자 기대 가능.
- 유료화 시기는 "너무 이르면 안 되고, 너무 늦으면 안 된다". 무료 사용자 1,000명 + 반복 사용자 100명 달성 후가 적절.
- 가격 앵커는 "시간 절약 가치"로 설정. 개발자 시급 기준 월 $19-49가 dev tool SaaS의 스위트 스팟.

   - 조직이 운영하는 모든 API/MCP 서버를 한 번에 스캔
   - CSV 업로드 -> 결과 대시보드
   - **대상**: 여러 마이크로서비스를 운영하는 팀

3. **모니터링 + 알림**
   - 주기적 자동 재스캔 (일/주)
   - 점수 변동 시 Slack/이메일 알림
   - "경쟁 도구 점수가 올랐다" 알림
   - **대상**: 지속적 품질 관리가 필요한 팀

   - GitHub Actions에서 `clarvia scan` 실행
   - PR 머지 전 AEO 점수 체크 (린트처럼)
   - 점수 하락 시 빌드 실패 옵션
   - **대상**: 품질 게이트를 자동화하려는 팀

5. **API 액세스**
   - 프로그래매틱 스캔, 결과 조회, 비교
   - 자체 대시보드/워크플로에 통합
   - **가격**: 호출당 과금 또는 월 한도

6. **화이트라벨**
   - 에이전시/컨설팅 회사가 고객에게 자체 브랜드 리포트 제공
   - **가격**: Enterprise 플랜 내
