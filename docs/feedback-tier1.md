## 1. MCP 서버 개발자

### 페르소나
**김재혁 (31세)** | 1인 오픈소스 개발자
GitHub에서 MCP 서버 3개를 운영 중. 하나는 Notion 연동, 하나는 Slack 연동, 하나는 PostgreSQL 쿼리용. 주간 다운로드 합계 약 2,000. 풀타임 직장(백엔드 개발자) 다니면서 퇴근 후/주말에 유지보수. Star 수와 npm 다운로드가 유일한 성과 지표.

### Clarvia를 왜 쓰겠는가
내 MCP 서버가 Claude나 Cursor 같은 에이전트에서 실제로 잘 발견되는지 궁금함. README에 설명은 잘 썼다고 생각하는데, 에이전트가 내 서버를 추천 목록에 올리려면 뭘 더 해야 하는지 모르겠음. 경쟁 서버(같은 카테고리)랑 비교해서 내가 뭐가 부족한지 알고 싶음.

### 좋은 점
1. **AEO 점수가 구체적이다.** 5가지 차원으로 나눠서 보여주니까 "아 내가 데이터 구조화는 잘 했는데 API 접근성이 떨어지는구나" 식으로 뭘 고쳐야 하는지 방향이 잡힌다. 그냥 "당신의 점수는 72점" 이러면 쓸모없었을 텐데.
2. **도구 디렉토리에 12,800개.** 내 서버도 등록할 수 있으니까 일단 노출 채널 하나가 느는 셈. 서비스 등록 기능이 있어서 직접 올릴 수 있는 것도 좋음.
3. **배지.** README에 AEO Score 배지 붙이는 건 꽤 영리하다. Star 수 배지 옆에 AEO 점수 배지 붙여놓으면 신뢰감 줄 수 있음.

### 불만/아쉬운 점
1. **점수 올리는 방법이 불명확하다.** 스캔 결과를 보면 "에이전트 호환성: 58점" 이라고만 뜨는데, 구체적으로 내 `tools/list` 엔드포인트의 어떤 부분이 문제인지, 스키마 설명이 부족한 건지, 에러 핸들링이 없어서인지 액션 아이템이 안 보인다. "이 점수를 85로 올리려면 이것 3개를 고쳐라" 같은 처방전이 필요함.
2. **MCP 서버 특화 메트릭이 없다.** API 일반이랑 MCP 서버는 다른데 같은 잣대로 측정하는 느낌. MCP는 tool description 품질, 파라미터 스키마 완성도, 에러 메시지 명확성 같은 게 중요한데 이런 세부 항목이 없음.
3. **경쟁 서버 비교가 수동이다.** 도구 비교에서 최대 4개까지 비교할 수 있다는데, 내 서버와 같은 카테고리의 상위 서버를 자동으로 벤치마킹해주면 훨씬 나을 텐데. 일일이 검색해서 찾아야 함.

### 킬러 피처
**"AEO 점수 변동 알림 + 개선 가이드."** 내가 README를 수정하거나 새 버전을 배포했을 때 자동으로 재스캔해서 "점수 72 -> 81, 이 변경이 효과적이었음" 알림을 보내주면 매일 체크할 이유가 생긴다. GitHub Actions 연동이면 완벽.

### 점수: 5/10
점수 보여주는 건 좋은데, 그 다음 행동으로 연결이 안 됨. 한번 스캔하고 "음 그렇구나" 하고 끝남. 반복 방문 동기가 약함.

## 2. API 프로덕트 매니저

### 페르소나
**이수진 (36세)** | PM, 핀테크 SaaS "페이링크" (직원 120명)
결제 API를 외부 개발자에게 제공하는 제품 담당. 월 API 호출 약 2억 건. Stripe, Toss Payments와 경쟁. 최근 CEO가 "AI 에이전트 시대에 우리 API가 에이전트에서 선택받으려면 뭘 해야 하냐"고 물어봤는데 대답을 못함. 이 질문에 대한 답을 찾고 있음.

### Clarvia를 왜 쓰겠는가
CEO한테 "AI 에이전트 최적화 현황 보고서" 올려야 하는데 근거 자료가 필요함. 우리 API 문서 URL 넣고 스캔해서 현재 점수 확인하고, Stripe 같은 경쟁사도 스캔해서 비교표 만들 생각.

### 좋은 점
1. **경쟁사 비교가 가능하다.** 우리 API 76점, Stripe 91점, Toss 82점 — 이런 식으로 보여주면 경영진한테 "우리가 여기서 뒤처져 있다"를 숫자로 설명할 수 있다. 내가 찾던 게 정확히 이거.
2. **5가지 차원 분해가 보고서에 바로 쓸 수 있다.** "우리는 데이터 구조화 90점인데 에이전트 호환성 61점이니까 에이전트 호환성에 투자해야 합니다" — 이 문장 하나로 예산 요청 근거가 됨.
3. **트렌딩/리더보드로 시장 흐름 파악.** 결제 카테고리에서 어떤 API가 올라오고 있는지 주간 단위로 볼 수 있으면 경쟁 모니터링에 유용.

### 불만/아쉬운 점
1. **PDF 리포트가 아직 안 됨.** "유료 기능 예정"이라는데, 이게 없으면 스크린샷 찍어서 파워포인트에 붙여야 함. 경영진 보고용으로 깔끔한 PDF가 당장 필요한데. 이거 언제 나옴?
2. **히스토리/트렌드가 없다.** 오늘 스캔하면 오늘 점수만 나옴. 우리가 API 문서 개선한 후에 점수가 올라갔는지 추적이 안 됨. "3개월간 우리 AEO 점수 변화" 그래프가 없으면 개선 효과를 증명할 수 없다.
3. **팀 기능이 없다.** 나 혼자 쓰는 게 아니라 개발팀, 문서팀이랑 같이 봐야 하는데 로그인/대시보드 공유 같은 게 안 보임. URL 공유밖에 안 되나?

### 킬러 피처
**"경쟁사 AEO 모니터링 대시보드."** 우리 API + 경쟁사 3개를 등록해놓으면 주간으로 자동 스캔해서 점수 변화를 추적해주는 대시보드. Stripe이 문서 업데이트해서 점수 올리면 우리한테 알림 오는 것. 이거 있으면 월 $50 낼 의향 있음.

### 점수: 6/10
보고서 한 번 만드는 데는 쓸 수 있는데, 지속적으로 쓸 이유가 부족. 원샷 도구 느낌. 대시보드+히스토리+팀 공유가 붙으면 7-8은 갈 수 있음.

## 3. DevRel / Developer Advocate

### 페르소나
**박민서 (29세)** | Developer Advocate, "코드플로우" (개발자 도구 스타트업, 30명)
CI/CD 파이프라인 도구를 만드는 회사에서 DevRel 담당. 블로그 월 4편, 유튜브 월 2편, 컨퍼런스 연 6회 발표. 개발자 커뮤니티 Discord 3,000명 운영. 최근 "AI 에이전트 호환성" 주제로 콘텐츠를 만들고 싶은데 앵글을 못 잡고 있음.

### Clarvia를 왜 쓰겠는가
"우리 CI/CD 도구의 AEO 점수를 공개하고, 점수 개선 과정을 블로그 시리즈로 만들겠다"는 콘텐츠 기획. 스캔 결과 스크린샷이 블로그 소재가 되고, 개선 전/후 비교가 유튜브 영상 소재가 됨. 배지를 README에 붙여서 "우리는 AI 에이전트 시대를 준비하고 있다"는 시그널도 보내고 싶음.

### 좋은 점
1. **콘텐츠 소재가 풍부하다.** "AEO 점수 47에서 89로 올리기" 같은 블로그를 쓸 수 있음. 5가지 차원별로 에피소드를 나눌 수도 있고. Before/After가 명확하니까 스토리텔링이 쉽다.
2. **배지 + SNS 공유 카드.** 도구 비교 결과를 공유 카드로 만들 수 있다는 게 좋음. 트위터에 "우리 도구 vs 경쟁사 AEO 비교" 올리면 개발자들이 반응할 것 같다.
3. **의도 기반 추천.** "I want to automate CI/CD" 검색하면 우리 도구가 나오는지 확인할 수 있음. SEO에서 키워드 체크하듯이 AEO에서 의도 체크를 할 수 있다는 개념이 신선함.

### 불만/아쉬운 점
1. **임베드가 안 된다.** 스캔 결과를 우리 문서 사이트나 블로그에 위젯으로 박고 싶은데 iframe이나 embed 코드가 없음. 스크린샷 찍어서 붙이면 업데이트가 안 반영됨.
2. **API가 없다.** 우리 CI/CD 파이프라인에서 빌드할 때마다 AEO 스캔 돌려서 점수 떨어지면 경고하게 만들고 싶은데, Clarvia 자체의 API가 없으니 자동화를 못함. 스캔 결과를 JSON으로 가져올 수 있어야 함.
3. **커뮤니티/코멘트가 없다.** 도구 페이지에 "이 도구 실제로 써봤는데 이런 점이 좋다/별로다" 같은 리뷰를 남길 곳이 없음. 우리 도구 페이지에 개발자 커뮤니티 피드백이 쌓이면 사회적 증명이 되는데.

### 킬러 피처
**"AEO 점수 변동 CI 체크."** GitHub Actions/GitLab CI에서 `clarvia scan --url https://docs.codeflow.dev` 하면 점수를 반환하고, 이전 대비 5점 이상 하락하면 PR에 코멘트 달아주는 것. 이거 있으면 블로그 소재도 되고 실제로 매일 파이프라인에서 돌릴 것.

### 점수: 5/10
콘텐츠 소재로 한두 번 쓰기엔 괜찮은데, 지속적 콘텐츠 루프를 만들려면 API, 임베드, CI 연동이 필수. 지금은 "스크린샷 찍어서 블로그 쓰기"가 최선인데 그건 금방 질림.

## 4. SaaS 창업자

### 페르소나
**정우성 (34세)** | CEO/CTO, "스케줄봇" (팀 5명)
AI 기반 미팅 스케줄링 SaaS. 아직 PMF 찾는 중. MRR $3K. Google Calendar, Slack 연동 API 있음. 최근 Calendly가 AI 에이전트 연동을 발표해서 위기감. "우리도 AI 에이전트에서 발견되어야 하는데 어디서부터 시작하지?"

### Clarvia를 왜 쓰겠는가
1) 우리 서비스가 AI 에이전트 관점에서 몇 점인지 현실 체크. 2) Calendly, Cal.com, SavvyCal 같은 경쟁사와 비교. 3) 빠르게 점수 올릴 수 있는 우선순위 액션 찾기. 초기 스타트업이라 리소스가 한정되니까 ROI 높은 것부터 하고 싶음.

### 좋은 점
1. **현실 직시 도구.** 우리가 38점이고 Calendly가 87점이면 "아 이 정도 격차구나"를 숫자로 보는 게 아프지만 도움됨. 막연한 불안감보다 구체적인 격차가 나음.
2. **도구 디렉토리 등록이 무료.** 스타트업 입장에서 노출 채널 하나가 공짜로 느는 건 무조건 좋다. SEO 하듯이 AEO 디렉토리에 올리는 것.
3. **의도 기반 추천에서 우리가 나오는지 확인.** "schedule a meeting automatically" 검색하면 Calendly만 나오고 우리는 안 나옴. 이걸 알면 뭘 고쳐야 하는지 역추적이 가능.

### 불만/아쉬운 점
1. **"그래서 뭘 하라는 건데?"가 없다.** 점수 보여주고 끝임. 초기 스타트업은 "1주일 안에 점수 20점 올리는 가이드" 같은 게 필요함. 항목별 우선순위 + 예상 소요시간 + 기대 점수 상승폭. 지금은 점수만 보고 멍해짐.
2. **온체인 보너스가 뜬금없다.** 우리는 Web3 서비스가 아닌데 왜 온체인 보너스가 점수 차원에 있는 거? SaaS 일반에는 해당 없는 기준이 끼어 있으니까 우리 점수가 불리하게 나오는 건 아닌지 의심됨. 카테고리별 가중치가 달라야 하지 않나.
3. **무료/유료 경계가 불명확하다.** PDF 리포트가 유료 예정이라고 하는데, 앞으로 뭐가 무료고 뭐가 유료인지 모르겠음. 갑자기 "이제 스캔 3회 이상은 유료입니다" 하면 곤란함. 초기 스타트업은 가격 예측 가능성이 중요.

### 킬러 피처
**"AEO 개선 플레이북 자동 생성."** 스캔 결과를 기반으로 "당신의 서비스에 맞는 3주 개선 플랜" 같은 걸 자동으로 만들어주는 것. 1주차: API 문서에 JSON-LD 추가 (예상 +12점), 2주차: OpenAPI 스펙 보강 (예상 +8점), 3주차: MCP 서버 만들기 (예상 +15점). 이거 있으면 바로 실행에 옮길 수 있고, 매주 진행 상황 체크하러 재방문함.

### 점수: 4/10
솔직히 한번 스캔하고 "아 우리 낮구나" 확인하고 나면 다시 올 이유가 없음. 점수 진단은 되는데 처방이 없으니까 병원에서 "당신 아프네요" 하고 약은 안 주는 느낌.

### 평균 점수: 5.0/10

### 핵심 인사이트
1. **진단은 되는데 처방이 없다** — 4명 모두 "점수를 보여줬으면 그 다음에 뭘 하라는 것도 알려줘"를 다른 표현으로 말함. 이게 가장 큰 갭.
2. **원샷 도구 → 반복 방문 도구 전환 필요** — 지금은 한번 스캔하고 끝. 히스토리, 알림, 모니터링이 붙어야 매일/매주 오는 도구가 됨.
3. **온체인 보너스 차원이 범용성을 해침** — Web3가 아닌 유저에게는 이질적. 카테고리별 가중치 조정이 필요.
4. **무료 → 유료 전환 시 가격 투명성** — 유료 기능이 뭔지, 얼마인지 미리 알려줘야 초기 유저가 안심하고 씀.
